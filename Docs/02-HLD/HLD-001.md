## 1. Purpose

This High-Level Design defines the logical architecture and major technology decisions for the RC-001 enterprise network.

It translates the requirements defined in RC001-NRS-001 into an implementable architecture without specifying every CLI command or interface-level configuration.

Detailed implementation information is defined in RC001-LLD-001.

---

## 2. Design Objectives

The architecture is designed to provide:

* Reliable multi-site communication
* Structured departmental segmentation
* Dynamic routing
* Centralized enterprise services
* Secure infrastructure administration
* Logical separation of management traffic
* Straightforward troubleshooting
* Modular future expansion
* Clear documentation and reproducibility

---

## 3. Enterprise Architecture Overview

Ashanti Logistics Ghana Ltd. consists of:

* Kumasi Headquarters
* Accra Branch
* Takoradi Branch

Kumasi operates as the central enterprise site.

It hosts:

* Core routing functionality
* Headquarters switching
* Centralized servers
* Centralized network-management connectivity
* WAN termination for branch connectivity

Accra and Takoradi use repeatable branch architectures consisting of:

* One router
* One managed switch
* User VLAN
* Management VLAN
* Routed WAN connection to headquarters

## 5. Hierarchical Design Model

### 5.1 Headquarters

RC-001 uses a compact hierarchical architecture appropriate for a simulated small-to-medium enterprise.

The traditional three-tier campus model is logically represented through:

* Core/routing function — HQ-R1
* Distribution/access aggregation — HQ-SW1
* User/access switching — HQ-SW2

Because this is a Packet Tracer laboratory with limited scale, the design uses a collapsed architecture rather than deploying separate physical core and distribution switch pairs.

This provides the benefits of hierarchy without introducing unnecessary equipment.

### 5.2 Branches

Branches use a simplified two-device architecture:

* Branch router
* Access switch

The design is repeatable and can serve as a template for future branches.

---

## 6. Headquarters VLAN Architecture

The headquarters uses separate VLANs for major organizational functions.

| VLAN | Function               |
| ---: | ---------------------- |
|   10 | Human Resources        |
|   20 | Finance                |
|   30 | Operations             |
|   40 | Information Technology |
|   50 | Executive/Management   |
|   60 | Servers                |
|   99 | Network Management     |

Benefits include:

* Reduced broadcast-domain size
* Clear logical separation
* Easier troubleshooting
* Better IP-address organization
* Foundation for future ACL and firewall policies

Important limitation:

VLAN segmentation alone does not enforce complete access control between departments. Because RC-001 permits inter-VLAN routing without comprehensive ACL policy, strict authorization controls are deferred to later security projects.

---

## 7. Branch VLAN Architecture

Each branch uses:

* VLAN 10 — Users
* VLAN 99 — Management

This standardization simplifies:

* Deployment
* Troubleshooting
* Documentation
* Future branch replication

Although VLAN IDs are reused across branch sites, each site uses unique IP subnets.

---

## 8. Routing Architecture

### 8.1 Interior Routing Protocol

OSPF is selected as the enterprise Interior Gateway Protocol.

RC-001 uses:

* OSPF Process ID 1
* OSPF Area 0
* Single-area design

### 8.2 Rationale

OSPF was selected because it:

* Supports dynamic route learning
* Scales better than extensive static routing
* Supports standards-based multi-vendor environments
* Provides a pathway to multi-area design
* Supports faster adaptation to topology changes than manually maintained static routes

### 8.3 Future Evolution

A larger enterprise could evolve toward:

* Multiple OSPF areas
* Route summarization
* Redundant WAN paths
* Route filtering
* Authentication
* SD-WAN

These features are outside RC-001 scope.

---

## 9. WAN Architecture

The WAN uses point-to-point routed IPv4 links.

Topology:

```text
Accra Branch -------- HQ -------- Takoradi Branch
```

HQ functions as the hub.

Each branch has a dedicated routed WAN subnet.

This design provides:

* Simple troubleshooting
* Clear route ownership
* Easy OSPF adjacency formation
* Straightforward future extension

A single-link-per-branch design creates a known single point of failure.

High availability will be addressed in RC-002.

---

## 10. Inter-VLAN Routing

Router-on-a-Stick is used for VLAN gateway services where applicable.

IEEE 802.1Q subinterfaces provide:

* VLAN default gateways
* Inter-VLAN routing
* Routed access to the WAN

This approach is appropriate for RC-001 because:

* It is supported in Packet Tracer.
* It clearly demonstrates VLAN-to-Layer-3 integration.
* It avoids unnecessary multilayer switching complexity in the baseline project.

A production environment with larger traffic volumes might instead use Layer 3 switching with switched virtual interfaces.

---

## 11. Server Architecture

Centralized enterprise servers reside in headquarters VLAN 60.

Representative systems include:

* AD-SRV
* DNS-SRV
* FILE-SRV
* WEB-SRV
* BACKUP-SRV

Centralized placement simplifies:

* Administration
* Addressing
* Access by multiple sites
* Backup planning
* Future security-policy enforcement

Some application roles may be represented conceptually depending on Packet Tracer capabilities.

---

## 12. Management Architecture

VLAN 99 is reserved for network infrastructure management.

Managed devices receive IP addresses from management subnets.

Remote administration shall use:

* SSH Version 2
* Local authenticated accounts
* Encrypted credential storage where supported
* Management IP addressing

Telnet shall not be required.

Future enhancements may include:

* TACACS+
* RADIUS
* AAA
* Dedicated management firewalling
* Out-of-band management

---

## 13. Security Architecture

RC-001 applies security primarily through architecture and administrative hardening.

Controls include:

* VLAN-based segmentation
* Dedicated management VLANs
* SSHv2
* Disabled Telnet dependency
* Shutdown of unused access ports where practical
* Secure administrative credentials
* Authorized-access banners
* Structured device management

RC-001 does not claim to implement a complete Zero Trust or enterprise access-control architecture.

Future projects will add:

* ACLs
* Firewalls
* VPNs
* IDS/IPS
* AAA
* Zero Trust controls

---

## 14. IP Addressing Strategy

Private IPv4 addressing shall use a structured enterprise hierarchy.

Conceptual allocation:

* `10.10.0.0/16` — Kumasi Headquarters
* `10.20.0.0/16` — Accra Branch
* `10.30.0.0/16` — Takoradi Branch
* `10.255.0.0/16` — WAN infrastructure

This site-based structure makes addresses easy to identify and supports future growth.

Detailed subnets are defined in RC001-LLD-001 and RC001-IPP-001.

## 16. Availability Considerations

RC-001 prioritizes architectural clarity over redundancy.

Known single points of failure include:

* HQ-R1
* Single branch WAN links
* Individual access switches
* Single inter-switch uplinks

These are intentional limitations.

RC-002 will investigate high-availability mechanisms such as:

* HSRP
* Redundant links
* EtherChannel
* Spanning Tree optimization
* Dual WAN paths

---

## 17. Scalability Strategy

The network is designed to scale through:

### Modular Site Addressing

Each site receives a dedicated address block.

### Repeatable Branch Design

Future branches can reuse the same logical structure.

### Dynamic Routing

OSPF reduces dependence on manually maintained static routes.

### VLAN-Based Segmentation

New departments can be added using additional VLANs and subnets.

### Hierarchical Design

The headquarters architecture can be expanded with additional access switches or upgraded to Layer 3 switching.

## 18. Design Decisions

| ------------------------- | ----------------------------------------------------------- |

| VLAN 99                   | Separates management addressing from users                  |
| Centralized servers       | Simplifies administration and shared-service access         |
| Site-based IP blocks      | Improves scalability and troubleshooting                    |

## 20. Key Risks

### Single Points of Failure

Accepted for RC-001 baseline.

### Misconfigured VLANs

Mitigated through standardized VLAN IDs and validation.

### Routing Misconfiguration

Mitigated through OSPF testing and route verification.

### Unauthorized Device Administration

Reduced through SSH and credential protection.

### Design Drift

Mitigated by maintaining alignment among NRS, HLD, LLD, diagrams, and final implementation.

---

## 21. Success Criteria

The high-level design is successfully realized when:

* The implemented network matches the approved architecture.
* Headquarters and branches exchange routes through OSPF.
* VLANs operate as designed.
* Centralized servers are reachable as required.
* Management addressing functions.
* SSH administration succeeds.
* Test evidence is documented.
* Deviations are recorded rather than silently introduced.
